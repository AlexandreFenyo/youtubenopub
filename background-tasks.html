Captured — Opérations background par type d'objet

Tableau récapitulatif des tâches lancées en arrière-plan, des mécanismes anti-collision en place, et des points de friction identifiés.

Déclencheurs d'opérations background

1. Actions explicites de l'utilisateur

  • 1APull-to-refresh sur la liste principale → pullToRefreshDates() : lève la pause + retry des aperçus en échec + retry du reverse geocoding pour les photos visibles sans placeName + retry du fetch <title> HTML pour les URLs visibles sans titre + retry des traductions Apple non encore appliquées + retry des OCR Vision en échec (sans relancer la description IA) + retry des transcriptions audio cancellées avant complétion + refresh batch des dates Last-Modified pour les URLs visibles.
  • 1BMenu …  ➜  Refresh previews and URL dates → toggleRefreshAllDates() branche start : équivalent au pull-to-refresh (inclut tous les retries listés en 1A).
  • 1CMenu …  ➜  Stop refreshing previews and URL dates → cancelAllRunningTasks() : annule TOUTES les tâches en vol et active autoTriggersPaused.
  • 1DMenu …  ➜  Refresh AI text → refreshAITextForCurrent() : reset titre/aiDescribed/ocrDone des photos/videos/audios visibles + lance explicitement les tâches IA pour chacun.
  • 1EMenu …  ➜  Reset AI counters → remet à zéro les compteurs globaux et les caps per-item de 5. Ne lance aucune tâche directement, mais débloque les triggers auto pour les items précédemment cappés.
  • 1FMenu trash  ➜  Clear previews of these URLs → clearPreviewsForCurrent() : annule les Tasks de génération en vol pour ces URLs, supprime les PNG, marque previewPath = "". Ne relance rien.
  • 1GMenu trash  ➜  Clear AI-generated text of these items → clearAITextForCurrent() : reset titre, marque aiDescribed = ocrDone = true pour bloquer relance auto. Ne relance rien.
  • 1HMenu contextuel d'une ligne  ➜  Regenerate preview (photo/video/file) → regeneratePreviewOnly(for:) : reset previewPath + lance explicitement startPreviewGeneration pour cet item.
  • 1IMenu contextuel d'une ligne URL  ➜  Regenerate preview and date → regeneratePreview(for:) : reset previewPath + modifiedAt, lance startFetchLastModified ET startPreviewGeneration en parallèle pour cet item.
  • 1JMenu contextuel d'une ligne  ➜  Regenerate AI text (photo/video/audio) → regenerateAIText(for:) : reset titre + marqueurs IA, lance startAIDescribe ou startAudioTranscription pour cet item.
  • 1KMenu contextuel d'une ligne IA-cappée  ➜  Reset AI call limit → remet aiCallsCount = 0 et aiDescribed = nil pour cet item, ce qui le rend à nouveau éligible aux triggers auto.

2. Événements système / cycle de vie

  • 2APartage d'un nouvel item via la Share Extension : le Share Extension écrit dans UserDefaults App Group et appelle WidgetCenter.reloadAllTimelines(). L'app principale détecte le nouvel item au prochain tick du timer 100 ms.
  • 2BDétection d'un nouvel item URL dans loadItems() (uniquement après le premier chargement, donc pas au tout démarrage de l'app) → startFetchLastModified(for:) automatique pour fetcher la date Last-Modified.
  • 2CReprise foreground (UIApplication.willEnterForegroundNotification / didBecomeActiveNotification) → loadItems() qui peut détecter de nouveaux items partagés pendant que l'app était en background.
  • 2DRender d'une ligne URL (.onAppear) → fetchTitle(for:) : récupère le <title> du HTML si l'item n'en a pas. Désormais tracé dans titleFetchTasks (anti-doublon, comptabilisé, annulable par 1C, timeout 10 s explicite).
  • 2ERender d'une ligne photo avec EXIF GPS sans placeName (.onAppear) → startReverseGeocode(for:) via CLGeocoder.
  • 2FDémarrage de l'app (.onAppear de ContentView) : aucun refresh automatique. L'utilisateur doit déclencher explicitement.

3. Triggers automatiques périodiques (timer 100 ms, sauf si autoTriggersPaused)

Important : il n'y a pas une tâche périodique par item. Il y a un seul timer global de 100 ms (le même que 5B) qui appelle loadItems(), lequel appelle ensuite les 4 fonctions triggerPending… ci-dessous. Chacune fait un balayage linéaire de tout le tableau items avec un for ... where ..., et ne démarre une Task.detached que pour les items qui passent le filtre ET qui ne sont pas déjà en cours de traitement (vérifié via les sets generatingPreviewIDs / describingIDs / fetchingDateIDs).
Timer 100 ms (1 seul) │ └─→ loadItems() │ ├─→ triggerPendingAIDescriptions() ── for item in items where … { startAIDescribe(item) } ├─→ triggerPendingOCR() ── for item in items where … { startOCR(item) } ├─→ triggerPendingAudioTranscriptions() ── for item in items where … { startAudioTranscription(item) } └─→ triggerPendingPreviews() ── for item in items where … { startPreviewGeneration(item) }
Coût d'un tick : 4 itérations linéaires sur items toutes les 100 ms (10×/s). Avec N = 1000 items, 40 000 itérations/s — négligeable côté CPU (les where ne lisent que des champs déjà en mémoire).
Latence d'un nouveau partage : entre 0 et 100 ms avant qu'une tâche ne démarre pour le nouvel item.
Effet de Stop refreshing : bascule autoTriggersPaused = true ; les 4 fonctions retournent alors immédiatement (guard !autoTriggersPaused else { return }) → plus aucune nouvelle tâche n'est démarrée. Les tâches déjà en vol continuent (annulées séparément par cancelAllRunningTasks).
  • 3AtriggerPendingAIDescriptions() : pour chaque photo/video sans aiDescribed et sous le cap per-item de 5 → lance startAIDescribe. Le cap (aiCallsCount) et le compteur global rate-limit sont incrémentés après complétion de la Task (et seulement si elle n'a pas été cancellée), pour que des Stop refreshing répétés ne consomment pas de crédit à tort.
  • 3BtriggerPendingOCR() : pour chaque photo/video avec aiDescribed == true (ou IA désactivée), sans ocrDone, et avec ocrFailed != true → lance startOCR. La Task est désormais tracée dans ocrTasks (annulable par 1C, comptabilisée), et les échecs sont mémorisés via ocrFailed = true pour bloquer les retries auto. Les refresh globaux 1A/1B remettent ocrFailed à nil pour retenter sans relancer la description IA.
  • 3CtriggerPendingAudioTranscriptions() : pour chaque audio sans aiDescribed → lance startAudioTranscription.
  • 3DtriggerPendingPreviews() : pour chaque item non-text avec previewPath == nil → lance startPreviewGeneration.

4. Effets de bord (déclenchent en cascade)

  • 4ARéception d'un Last-Modified plus récent que celui qu'on avait dans updateItemModifiedAt → previewPath de l'URL repassé à nil → triggerPendingPreviews au prochain tick recapture la vignette.
  • 4BTap sur une ligne → markAsSeen(itemID:) qui met à jour lastSeenModifiedAt (pas une tâche background, mais persiste).
  • 4CReception réussie d'une description IA Apple alors que la langue de l'app n'est pas anglais → ajoute un job dans pendingLabelTranslations que LabelTranslator draine via TranslationSession. Les jobs en queue sont désormais comptabilisés dans le compteur Background tasks et annulables par 1C ; le drapeau translationDone stocké sur chaque item bloque les retries auto et permet aux refresh globaux (1A/1B) de relancer la traduction des items qui ne l'ont pas reçue.

5. Périodiques non liés aux items

  • 5ATimer 2 s dans StatsPanelView → recomputeAppDataBytes() : traversée récursive du container App Group pour mettre à jour la taille affichée.
  • 5BTimer 100 ms global dans ContentView → loadItems + checkAIEnabledTransition + checkShareCountForReview.

Mécanismes anti-collision en place

  • generatingPreviewIDs : bloque le lancement parallèle d'une 2ᵉ génération de vignette pour le même item.
  • describingIDs : bloque la 2ᵉ description IA / transcription audio pour le même item.
  • fetchingDateIDs : bloque le 2ᵉ fetch Last-Modified simultané pour le même item.
  • geocodingIDs : bloque le 2ᵉ reverse geocoding pour le même item.
  • WebPageRenderSemaphore : sérialise globalement les captures WKWebView (une seule à la fois pour toute l'app).
  • PreviewGenerator.blankCanvas dispatché sur le main thread pour les rendus UIKit.
  • autoTriggersPaused : pause sticky des 4 trigger functions après un Stop refreshing.
  • Task.isCancelled vérifié dans renderWebPage aux 3 points-clés (avant acquire, après acquire, après load) pour drainer rapidement les Tasks annulées.
  • Dictionnaires previewTasks / aiTasks / dateFetchTasks / geocodeTasks : Task<Void, Never> indexés par id pour annulation explicite.
  • OCR ↔ IA : triggerPendingOCR attend explicitement que aiDescribed == true (ou que l'IA soit désactivée) avant de lancer.
